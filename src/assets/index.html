Sessions

Group a run of commands and their outputs under a single session — so an AI agent like Claude can link a working session to the exact databases and tables it produced, and you can replay what happened.

The plan:

  • Tag each command with a session id (e.g. from MUCKDB_SESSION) so related work is grouped.
  • Show each session as a timeline of commands with the databases and tables it touched.
  • Let an agent open a session and jump straight to the outputs it created.
